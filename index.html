Kraken

A lightweight, mobile-first boilerplate for front-end web developers.

Kraken includes just the essentials:

  • A CSS reset to ensure cross-browser compatibility.
  • A responsive, mobile-first grid.
  • A well-designed, fluid typographic scale.
  • CSS3 buttons.
  • Basic form styling.
  • Separate add-ons for additional functionality.

The Grid

Kraken uses Origami, a fluid, mobile-first grid system.

It features a fluid 6-column grid that can scale to any size. It includes offsets, grid nesting, and special classes for content choreography and image galleries.

Typography

Kraken uses relative sizing (ems and %, not pixels) for everything.

The body has a base font-size of 106.25%, which is 17px on browsers with default font settings. All other sizes are in ems. Changing the font-size on the body element will adjust the typographical scale for the entire site.

The font-family on the body is "Helvetica Neue", Arial, sans-serif. This sets the typeface for the entire site. Adjust as needed.

The typographic scale was built using Iain Lamb's Typograph app.

Semantic Heading Classes

All heading values can also be applied as classes. For example, if a heading should be an h1 element semantically, but you would like it to be styled as an h4 element, you can apply class="h4" to the h1 element.

This is an h1 heading that's styled as an h4 heading

<h1 class="h4">This is an h1 heading that's styled as an h4 heading</h1>

Buttons

Button styles can be applied to anything with the .btn class applied. However, typically you'll want to apply these to only <a>, <button>, and <input type="submit"> elements for the best rendering.

<button class="btn">Button</button>
<button class="btn btn-color">Button Color</button>
<button class="btn btn-large">Button Large</button>


<button class="btn btn-color btn-block">Button Block</button>
<button class="btn btn-block">Button Block</button>

Forms

Labels, legends and inputs are styled as full width block elements (with the exception of checkboxes and radio buttons). Use the grid system to control their sizing and alignment.

<form>
    <label>Label</label>
    <input type="text">
    <label>
        <input type="checkbox">
        Option 1
    </label>
    <label>
        <input type="checkbox">
        Option 2
    </label>
    <label>
        <input type="radio" name="radioset">
        Option 1
    </label>
    <label>
        <input type="radio" name="radioset">
        Option 2
    </label>
    <select>
        <option>Select 1</option>
        <option>Select 2</option>
        <option>Select 3</option>
    </select>
    <textarea></textarea>
</form>

Pro Tip: Wrap checkboxes and radio buttons inside a <label></label> to make them easier to click.

Screen Readers

If you have text that you don't want displayed on screen, but that should still be in the HTML for screen readers (for example, a search form label), simply apply the .screen-reader class. This will shift content off-screen, but screen readers can still read it.

<form>
    <label class="screen-reader">Search this site</label>
    <input type="text" placeholder="Search this site...">
    <input type="submit">
</form>

Add-Ons

Kraken was built to be lightweight but expandable. Here are some add-ons that were designed to work with Kraken:

  • Kraken for WordPress, a WordPress theme boilerplate.
  • Icon fonts, a lightweight, scalable way to include icons.
  • Astro, a collection of mobile-first navigation patterns.
  • Drop, mobile-friendly dropdown menus.
  • Houdini, a simple expand-and-collapse widget.
  • Tabby, lightweight, mobile-first toggle tabs.
  • Snapshot, basic image styling.
  • Tables, simple CSS table styling.
  • FlexSlider, a responsive, touch-enabled image carousel.
  • .htaccess, best practice server rules for making web pages fast and secure.

Browser Support

Kraken works on all major browsers.

Because IE 8 and lower do not recognize media queries, they will receive the single-column mobile layout instead of the fluid grid.

Also, rounded corners, gradients and drop-shadows on things like buttons and forms aren't recognized by IE 8 and lower and some mobile browsers (like Opera Mini). Forms and buttons still work in those browsers - they just look a bit simpler.

Viewport Sizing

Don't forget to include viewport sizing in the <head> for mobile devices.

<head>
    <meta name="viewport" content="width=device-width">
    ...
</head>

HTML5 Shim

If you plan on using the new HTML5 semantic elements, you should also include the HTML5 Shim in the <head> for older browser support.

<!-- HTML5 Shim for IE 6-8 -->
<!--[if lt IE 9]>
    <script src="http://html5shiv.googlecode.com/svn/trunk/html5.js"></script>
<![endif]-->
